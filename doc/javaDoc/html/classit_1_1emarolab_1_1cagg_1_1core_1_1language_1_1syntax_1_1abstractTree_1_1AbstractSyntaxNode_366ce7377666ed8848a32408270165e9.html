This class extends it.emarolab.cagg.core.language.SyntaxNode, where the type

T

is it.emarolab.cagg.core.language.syntax.abstractTree.AbstractSyntaxNode.Type, while the data

D

are classes that extends it.emarolab.cagg.core.language.BaseData (typically instances of: it.emarolab.cagg.core.language.syntax.abstractTree.AbstractDataFactory.ASNodeEmptyData, it.emarolab.cagg.core.language.syntax.abstractTree.AbstractDataFactory.ASNodeStrData, it.emarolab.cagg.core.language.syntax.abstractTree.AbstractDataFactory.ASNodeStrStrData or it.emarolab.cagg.core.language.syntax.abstractTree.AbstractDataFactory.ASNodeStrTreeData. Instantiated through it.emarolab.cagg.core.language.syntax.abstractTree.AbstractDataFactory).

This is done in order to implement a generic Note (ASN) for the Abstract Syntax Tree (AST). This kind of tree is build from the Parsing Tree (PS) and it only contains the minimal set of information. Particularly, the AST contains a root node with two children. The first collects (at the same depth) all the preamble of the source code, while the second contains, in the same structure, all the rules.

Those nodes (so the AST) are instantiated during the parsing tree building performed by: it.emarolab.cagg.core.language.parser.ParserListener; activable during the constructor of the object: it.emarolab.cagg.core.language.parser.TextualParser.

Last but not the least, the final implementation of those type of nodes are collected in the package: { dotVocal.nuanceGoogleIntegration.semanticParser.parser.abstractSyntaxGenerator.nuanceSyntaxNode.syntaxNodeType}.
