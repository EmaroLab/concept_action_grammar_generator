This is the base class for all the nodes in all the trees of the system in order to assure a unique identifier along all the structure (where, the ID is implemented as an incremental

Long

number.

Each nodes are described in terms of

Type T

,

Data D

. Where,

T

is usually an enumerator for easy node type definition, while

D

is a generic Object templated in the system classes structures for visualisation as well as maintenance purposes.

At the same time, this class implements the procedures to print iteratively the tree and to copy it with the possibility to have different node ID.

Last but not less important, the main classes that implements this object are: it.emarolab.cagg.core.language.syntax.abstractTree.AbstractSyntaxNode and it.emarolab.cagg.core.language.syntax.expressionTree.ExpressionNodeOld

it.emarolab.cagg.core.BaseNode.BaseNode ( )

Create a new tree node.
It does not call assign() so, it does not instantiate the

type

and

data

fields. That is why the non abstract implementation of this class must implement the abstract interfaces: assignData() and assignType(). Moreover, in order to fully initialise the node the non abstract implementation should call assign() in the constructor (considering also possible node coping).
It also initialises the node ID and sets DEFAULT_PREFIX_ID for printing it on debugging.
