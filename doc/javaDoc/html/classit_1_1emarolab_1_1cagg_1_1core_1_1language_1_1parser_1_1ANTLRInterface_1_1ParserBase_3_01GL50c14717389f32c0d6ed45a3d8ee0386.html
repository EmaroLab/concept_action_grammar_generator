This class implements a base structure to parse a source code in order to generate the Parsing Tree (PT); where the source text can be given as a String as well as a File (by specifying its directory path). Also, it is possible to connect to this class a Listener in order to walk through the PT and create (by node entrance/exit listening) the Abstract Syntax Tree (AST) (which contains also Expression Trees (ET)). More in particular, the class is designed in order to parse the file on constructor and so create the PT. Then, it uses the Listener (that can be also set through setListener(ParseTreeListener) or setListener(ParseTreeListener, ParserRuleContext)) in order to walk trough the PT and create the AST.

Moreover, this class must be based on the classes created by the ANTLR library, those are collected in the package:

dotVocal.nuanceGoogleIntegration.semanticParser.parser.contentsParser.ANTLRInterface.ANTLRGenerated

. In which are contained the base class types for all the parameters:

GLexer

,

GParser

,

GContext

and

GListener

. Furthermore, consider that those objects are generated starting from an ANTLR grammar definition located in this project in the folder:

../files/parserGrammar/NUanceGrammar.g4

.

Last but not the least, this class can be also used in order to show a simple GUI visualisation of the PS; which is completely independent from the GUI defined within the packages:

dotVocal.nuanceGoogleIntegration.semanticParser.debuggingGui

<GLexer>this is the Lexer used by the ANTL library to parse the source text. It must extends the Lexer class. Typically, for this system, it would be the CAGGSyntaxDefinitionLexer or an extension.
<GParser>this is the Parser used by the ANTL library to parse the source text. It must extends the Parser class. Typically, for this system, it would be the CAGGSyntaxDefinitionParser or an extension.
<GContext>this is the rule context of the grammar from which this object start parsing. It must extends the ParserRuleContext class. Typically, for this system, this is the main rule defined in the ANTL grammar.
<GListener>this is the class that by listening the creation of the PS creates and populates the AST. It must extends the ParseTreeListener class. Typically, for this system, it would be the ParserListener or an extension.
