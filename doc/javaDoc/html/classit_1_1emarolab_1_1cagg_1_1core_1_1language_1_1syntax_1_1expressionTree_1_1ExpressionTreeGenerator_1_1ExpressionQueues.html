This class implements and manage two different java.util.Queue of it.emarolab.cagg.core.language.syntax.expressionTree.ExpressionNodeOld. Particularly, one queue contains only the Operation nodes, while the other only the Term node.

Moreover this class monitor the state of the queues by considering the number of elements and, based on that, it returns the type of pending action to be performed in order to build the ET through the methods: hasPending(), hasOperatorPending() and hasTermPending().

Integer it.emarolab.cagg.core.language.syntax.expressionTree.ExpressionTreeGenerator.ExpressionQueues.hasPending ( )

This method check the state of the Operation and Term queues. Particolarly, it returns: ExpressionTreeGenerator#PENDING_NO if both queues are empty. Or: ExpressionTreeGenerator#PENDING_OPERATION if the Operation queue is not empty. Or: ExpressionTreeGenerator#PENDING_TERM if the Operation queue is empty but the Term queue not. Or: ExpressionTreeGenerator#PENDING_ERROR otherwise.

Returns
the actual pending node in queues tag.
